## 4.6、版本回退，并将回退后的代码推送到远程

```txt
//① 将版本进行--hard回退
git reset --hard 要回退到的版本号

//此时是无法将代码推送到远程的，因为远程最新的版本号在本地是不存在的
//此时要保证远程最新的版本号在本地是存在的，利用--mixed修改本地历史仓库的版本号

//② 利用--mixed操作，保证本地历史仓库版本号和远程一致
git reset --mixed 远程最新的版本号

//③ 将本地提交并推送
git add .
git commit -m 'xx'
git push -u origin master
